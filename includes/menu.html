Idea Summary
Area: Watson
Description: Code evolves over multiple releases/projects and we have lot of documents and code related to the projects. While designing for latest projects, we often consult experienced people and legacy documents to understand why something is introduced in someway. Similarly, if a customer raised PMR, it is common practice to consult people or refer to documents. If it is common or popular question, we can easily find it by a simple Google search or searching in PMR database. But if your goal is to find answers for unpopular questions, simple search is not effective. A study shows unpopular questions about a product will consume 80% of the time to retrieve an answer.
Lets take example of popular questions which we can easily obtain answers using a google search or searching in PMR database
  • What is the max transfer size for a CAPI disk?
  • Does Rains adapter support interrupts?
  • What is resetToggle bit?
Reflect on the following unpopular questions that arise for a development or a L3 team member:
  • Why was max tranfer size of CAPI disk increased from 128KB to 4GB?
  • Why we use timers to poll for transmit completions in Rains adapter?
  • What is the reason behind combining disk and adapter drivers for accessing CAPI disks?
  • Why resetToggle bit is checked for USB 2.0 devices but not for USB 3.0 devices?
  • What was the maximum transfer size of scsidisk when MPIO/PCM was not available?
The above unpopular questions will take lot of time to get answers. To solve such problems, we propose a system that uses Watson's service and helps delevopment/L3 teams to find answers for unpopular questions (the questions that cannot be solved using simple search) quickly.
Our proposed system contains three parts:
  • user Interface
  • Question and Answer Subsystem running on Server
The System works in the following way:
  • All FRS, Design Documents will be fed to Watson from Database. All the documents are converted in to JSON using "Document Conversion" Watson service.
  • The data is trained with sample questions and answers. This happens in Q&A subsystem.
  • User queries through UI and retrieves answers from the Q&A subsytem. Users summarize their question and submit it to the Retrieve and Rank Application service running in Q&A subsystem. The system searches the database, and generates, scores and ranks the answers. The system presents the most relevant answers to the user.
Future Enhancements
In addition to FRS/Design documents, increase the data by feeding from additional following data sources like
  • Slack channel data which is created when a new LI is being worked on
  • All mail data related to some lineitem.
  • Source Code and doxygen call graphs
Technical Information
The front end is implemented using php and the application is deployed in Bluemix. The Bluemix application interacts with watson services running on the cloud. We have used Document Conversion and Retrieve and Rank Watson services. Since we need to upload documents on cloud, we did not upload FRS and design documents. For POC purposes, we have used few documents on USB and RSCT to train our watson.
